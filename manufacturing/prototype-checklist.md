# 28mm PCB到着時チェックリスト

## 目視検査

- [ ] 基板の形状: 28mm円形か？
- [ ] パッド: nRF5340 QFN-94 (7x7mm) のパッドが正しいか？
- [ ] パッド: nRF21540 QFN-16 のパッドがあるか？
- [ ] USB-C: フットプリントが正しいか？
- [ ] チップアンテナ: パッドがあるか？
- [ ] SWDパッド: 4つのテストパッドがあるか？（SWDIO, SWDCLK, VCC, GND）
- [ ] GND除外ゾーン: アンテナ下に銅がないか？（裏面確認）
- [ ] シルク: 部品番号が読めるか？ピン1マークがあるか？
- [ ] ビア: すべてテンティングされているか？

## 寸法確認

- [ ] ノギスで直径測定: 28.0mm +/-0.1mm
- [ ] 厚さ: 1.0mm +/-0.1mm
- [ ] USB-C開口部: コネクタが物理的に入るか？

## フットプリント確認（部品を載せてみる）

- [ ] nRF5340 QFN-94: パッドにピンが合うか？（ルーペで確認）
- [ ] nRF21540 QFN-16: パッドにピンが合うか？
- [ ] USB-C: コネクタがパッドに合うか？
- [ ] MAX98357A: パッドにピンが合うか？
- [ ] 0402パッシブ: パッドサイズ確認（100nFを1個載せてみる）
- [ ] JST-PH-2P: コネクタがパッドに合うか？
- [ ] 水晶 32MHz (2012): パッドに合うか？
- [ ] 水晶 32.768kHz (1610): パッドに合うか？

## 電気的確認（はんだ前）

- [ ] VCCパッドとGNDパッド間に短絡がないか（テスターで確認）
- [ ] SWDパッド間に短絡がないか

## はんだ付け後の確認

- [ ] nRF5340周辺のブリッジがないか（顕微鏡で全ピン確認）
- [ ] nRF21540周辺のブリッジがないか
- [ ] USB-Cピンのブリッジがないか
- [ ] 3.3V LDO出力: AP2112K出力ピンで3.3V +/-0.1Vが出ているか
- [ ] 充電IC: USB-C給電時にTP4054が動作するか

## SWD接続テスト

- [ ] J-Link/nRF DKのSWDケーブルを接続
- [ ] `nrfjprog --ids` でデバイスが認識されるか
- [ ] `nrfjprog --memrd 0x00FF0000 --n 16` でFICR読み出し（nRF5340のDevice ID確認）
- [ ] `nrfjprog --eraseall` で全消去成功するか
- [ ] テストファームウェア（LED点滅）を書き込んで動作確認

## ファームウェア書き込み

- [ ] Auracast受信ファームウェアをwest buildでビルド
- [ ] `west flash` でnRF5340 cpuappに書き込み
- [ ] `west flash` でnRF5340 cpunetに書き込み（BLE controller）
- [ ] 電源ON後、Auracast broadcast sourceの音声が受信できるか

## 問題があったら

| 問題 | 対応 |
|------|------|
| パッドずれ | gen_coin_lite_v2.py を修正 -> 再発注 |
| 配線エラー | KiCadで再設計を検討 |
| QFNはんだ不良 | ホットエアで再加熱、フラックス追加 |
| SWD認識しない | VCC/GND確認、SWDIOプルアップ確認 |
| USB-C給電しない | CC抵抗(5.1k)確認、コネクタはんだ確認 |
| 3.3V出ない | AP2112K入出力確認、デカップリングC確認 |
