The Problem 課題

Traditional PA systems can't deliver synchronized audio to every seat in a stadium. Physics wins. 従来のPAシステムでは、スタジアム全席に同期した音声を届けることができません。物理法則には勝てない。

📡

WiFi Multicast Limit WiFiマルチキャストの限界

~500

WiFi multicast fails above ~500 devices. Packet loss becomes catastrophic. 55,000 is impossible. WiFiマルチキャストは約500台を超えるとパケットロスが致命的に。55,000台は不可能。

🔊

Speed of Sound 音速の壁

630ms

Sound travels 343 m/s. At Tokyo Dome (216m stage-to-back), the back row hears audio 630ms late. Nearly 2/3 of a second behind. 音速は343m/s。東京ドーム（ステージから最後列216m）では最後列が630ms遅れて聞こえる。

🏟️

PA Can't Compensate PAでは補正不可

216m

Delay towers help, but still can't eliminate the fundamental physics. Back rows always hear a delayed, muddy mess. ディレイタワーは改善するが根本解決にはならない。後列は常に遅延した不明瞭な音を聞く。

The Solution — 3-Layer Architecture ソリューション — 3層アーキテクチャ

Fiber backbone. 5G distribution. GPS-scheduled playback. Every seat, same moment. 光ファイバー基幹。5G配信。GPS同期再生。全席、同じ瞬間。

LAYER 1

BACKBONE — Fiber + Dante BACKBONE — 光ファイバー + Dante

The Koe Hub connects to a redundant Dante audio network over fiber optic ring. 48 zone processors receive audio with <1ms latency across the entire venue. Dual fiber paths ensure zero single-point-of-failure. Koe Hubは冗長化されたDanteオーディオネットワークに光ファイバーリングで接続。 48台のゾーンプロセッサが1ms未満のレイテンシで音声を受信。 デュアルファイバーでSPOFゼロ。

LAYER 2

DISTRIBUTION — 5G Broadcast + GPS DISTRIBUTION — 5Gブロードキャスト + GPS

Each zone processor drives 12 5G small cells via eMBMS (evolved Multimedia Broadcast Multicast Service). Each small cell multicasts to ~100 COINs. GPS 1PPS on every zone processor provides atomic clock synchronization. 576 small cells total cover the entire dome. 各ゾーンプロセッサがeMBMS経由で12台の5Gスモールセルを駆動。 各セルが約100台のCOINにマルチキャスト。GPS 1PPSが全ゾーンプロセッサに 原子時計同期を提供。576台のセルでドーム全体をカバー。

LAYER 3

CROWD — 55,000 COIN Lite + LED CROWD — 55,000 COIN Lite + LED

Each COIN Lite receives audio via 5G/LTE-M and syncs to GPS time. Playback is GPS-scheduled (not network-triggered) with per-zone delay compensation. LED commands are embedded in the audio stream metadata. The result: every seat hears the same beat at the same moment. 各COIN Liteが5G/LTE-M経由で音声受信、GPS時刻に同期。 再生はGPS同期スケジュール（ネットワークトリガーではない）でゾーン毎の遅延補正付き。 LEDコマンドは音声ストリームのメタデータに組み込み。全席が同じ瞬間に同じビートを聴く。

LED Orchestration LED オーケストレーション

55,000 synchronized LEDs. The world's largest crowd-powered light show. 55,000台の同期LED。世界最大の観客参加型ライトショー。

Show File Throughput ショーファイルスループット

3.3M cmd/sec

60fps LED commands x 55,000 devices = 3.3 million commands per second. Solved by zone-based addressing: 48 zones x 60fps = 2,880 commands/sec. 60fps LEDコマンド x 55,000台 = 毎秒330万コマンド。ゾーンベースで解決: 48ゾーン x 60fps = 2,880コマンド/秒。

Per-Device Effects デバイス単位エフェクト

hash(device_id) % period

Wave, ripple, and cascade effects within each zone. Each COIN applies a hash-based offset to the zone command, creating organic patterns without per-device addressing. ゾーン内のウェーブ、リプル、カスケード効果。各COINがデバイスハッシュに基づくオフセットを適用し、個別アドレス不要で有機的なパターンを生成。

Camera-Visible Patterns カメラ映えパターン

Spell words, draw images, create synchronized waves across the entire crowd. Zone-to-zone coordination creates stadium-wide visual patterns visible from broadcast cameras. 文字を表示、画像を描画、観客全体で同期ウェーブ。ゾーン間連携でスタジアム規模のビジュアルパターンを放送カメラから視認可能。

GPS-Timed Execution GPS同期実行

LED commands carry a GPS timestamp. All 55,000 COINs execute the pattern at the exact same microsecond. No visible propagation delay. LEDコマンドにGPSタイムスタンプを付与。全55,000台のCOINが同一マイクロ秒にパターンを実行。伝搬遅延ゼロ。

Redundancy — Zero Downtime 冗長性 — ダウンタイムゼロ

Stadium shows can't fail. Every component has a backup. スタジアム公演は停止できない。全コンポーネントにバックアップ。

Dual Fiber Ring デュアルファイバーリング

Primary + backup fiber paths. If one is cut, traffic reroutes in <10ms. The show continues. プライマリ + バックアップの光ファイバーパス。片方が切断されても10ms未満で再ルーティング。

Dual Zone Processors デュアルゾーンプロセッサ

Each zone processor has dual power and dual network uplinks. 4 spare units on standby for immediate swap. 各ゾーンプロセッサはデュアル電源 + デュアルネットワーク。予備4台がスタンバイ。

COIN Fallback COINフォールバック

If 5G fails, each COIN falls back to WiFi mesh mode (degraded quality, but audio continues). Graceful degradation. 5G障害時はWiFiメッシュにフォールバック（品質低下するが音声は継続）。グレースフルデグラデーション。

Hub Hot Standby ハブ ホットスタンバイ

Redundant Koe Hub pair. Hot standby takes over in <100ms. Audience won't notice. 冗長Koe Hubペア。ホットスタンバイが100ms未満でテイクオーバー。観客は気づかない。
