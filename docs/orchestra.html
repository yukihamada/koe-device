なぜ送信にUWBを使うのか

BLEはブロードキャスト受信には十分。
しかしライブ楽器ミキシングにはサブミリ秒の精度が必要です。UWBがその鍵になると考えています。 BLE is fine for broadcast (Rx), but live instrument mixing demands sub-millisecond precision.

BLEオーディオのレイテンシ

20-40ms

プリミックスされたブロードキャスト受信には許容範囲。しかしライブ楽器をミキサーに送信するには不適格 — 奏者は自分の音が遅れて聞こえてしまいます。

UWB (DW3000) のレイテンシ（理論値）

<1ms

DW3000は6.8Mbps、ナノ秒精度のタイミングで動作。48kHz/16bitモノラルの場合、1楽器に必要な帯域はわずか768kbps。サブマイクロ秒のクロック同期で高精度なミキシングが理論上可能です。

UWBの帯域容量

6.8 Mbps

各HubのDW3000はTDM（時分割多重）で約8チャンネルを処理可能と想定。4台のHubでフルオーケストラ100人分をカバーする構想です。位置情報データも付随 — Hubは各奏者の位置を把握できます。

Auracastブロードキャスト（受信）

30-50m

BLE Auracastはリターンパスに最適：1つのブロードキャストが全奏者に同時到達。ペアリング不要。接続数制限なし。DK版で体験可能。PA/LNA搭載版で距離拡大を計画中。

レイテンシバジェット（理論値）

7msはライブ演奏では感知不能レベルです。現行の有線システムは3-5ms。
現行のワイヤレスIEM（Shure PSM、Sennheiser）は受信のみで3-5ms — 送信はできません。
Koeは両方を目指します。 All latency figures are theoretical targets, not measured values.
