How it works

Three steps. Zero friction.

From presence in the room to a finished recording — the loop runs itself.

01
KAGI
KAGI detects presence
Using device pings, door sensors, and Wi-Fi triangulation, KAGI knows exactly who is home and in which room. This context is broadcast to the Shared Brain.
presence.detected person_count: 2 room: music_room
Yuki — home Guest — home Away
02
KOE
KOE detects instrument onset
Koe Stone's always-on mic monitors for the acoustic fingerprint of an instrument being picked up — a guitar strum, piano key, or drum hit. Onset is confirmed within 400 ms.
session.started instrument: acoustic_guitar confidence: 0.92
03
KOE + KAGI
Both apps act in sync
KAGI dims the lights to 18%, sets the thermostat to a comfortable recording temperature (22°C), and silences notifications. KOE begins multi-track recording with the correct number of tracks based on the people present in the room.
lights → 18% thermostat → 22°C DND: on recording: start tracks: 2
Technical

The Shared Brain

A lightweight event bus running locally on your network. Both apps speak the same JSON protocol. No cloud required.

Event Bus API
Either app posts events to the bus. All connected apps receive them instantly via WebSocket.
POST /api/v1/brain/event Send an event from KOE or KAGI 
{
  "source_app": "kagi",            // "koe" | "kagi"
  "event_type": "presence.detected",  // see event types below
  "payload": {
    "room": "music_room",
    "person_count": 2,
    "persons": ["yuki", "guest"]
  }
}

Event Types

Standard events understood by both apps. Extend with any custom payload fields.

presence.detected
Someone's home
Room, person count, and optional identities. Triggers track preparation.
Source: KAGI
session.started
Music begins
Onset confirmed. Includes detected instrument and confidence score.
Source: KOE
session.ended
Session over
Silence detected or manual stop. Triggers room restore — lights back, thermostat normal.
Source: KOE
morning_mix.ready
Morning preview
Koe has prepared a 60-second highlight from last night's best session.
Source: KOE
presence.left
Room empty
KAGI saw everyone leave. KOE can auto-pause or save unfinished sessions.
Source: KAGI
room.mode_change
Mode switched
Scene change (recording, movie, sleep). Both apps adapt their behaviour.
Source: Either
Privacy

All local. No cloud required.

The Shared Brain runs entirely within your home network. Audio never leaves the device; only structured events are exchanged.

🏠
On-device processing
Instrument onset and presence detection run on Koe Stone and KAGI's local hub respectively. No audio is uploaded.
🔒
Local event bus
The /api/v1/brain/event endpoint runs on your home server (or koe.live when cloud mode is enabled). LAN-first.
💬
Structured events only
Events contain room names, timestamps, and instrument labels — never raw audio data or personal identifiers.
