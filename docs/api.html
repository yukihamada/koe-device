Koe API

誰でも、どこでも、何千人にも音声を届けられるインフラ。
サードパーティ開発者がKoeの音声配信基盤の上にアプリを構築できます。

https://koe.live/api/v1

認証 Authentication

APIキー

すべてのリクエストに API キーをヘッダーに含めてください。

Request Header 
Authorization: Bearer YOUR_API_KEY
cURL Example 
curl https://koe.live/api/v1/broadcast/list \
  -H "Authorization: Bearer YOUR_API_KEY"

APIキーは ダッシュボード から発行できます。キーは安全に保管し、クライアントサイドのコードに含めないでください。

配信管理 Broadcast

POST /api/v1/broadcast/start

音声配信を開始します。チャンネル名、コーデック、サンプルレートを指定してください。

Request Body 
{
  "channel_name": "メインステージ",
  "codec": "lc3",
  "sample_rate": 48000
}
Response — 200 OK 
{
  "broadcast_id": "bc_xxx",
  "status": "active"
}
POST /api/v1/broadcast/stop

配信を停止します。

Request Body 
{
  "broadcast_id": "bc_xxx"
}
Response — 200 OK 
{
  "broadcast_id": "bc_xxx",
  "status": "stopped"
}
GET /api/v1/broadcast/list

アクティブな配信一覧を取得します。

Response — 200 OK 
{
  "broadcasts": [
    {
      "broadcast_id": "bc_001",
      "channel_name": "メインステージ",
      "codec": "lc3",
      "listeners": 847,
      "started_at": "2026-07-15T12:00:00Z"
    }
  ]
}

デバイス管理 Devices

GET /api/v1/devices

接続中のデバイス一覧を取得します。

Response — 200 OK 
{
  "devices": [
    {
      "id": "SEED-0001",
      "battery": 87,
      "status": "active",
      "channel": "メインステージ"
    }
  ]
}
GET /api/v1/devices/:id

指定デバイスの詳細情報を取得します。バッテリー残量、ファームウェアバージョン、接続チャンネル、最終通信時刻などを含みます。

Response — 200 OK 
{
  "id": "SEED-0001",
  "battery": 87,
  "status": "active",
  "channel": "メインステージ",
  "firmware_version": "1.2.0",
  "last_seen": "2026-07-15T14:30:00Z"
}
POST /api/v1/devices/:id/locate

デバイスの位置を取得します（Find My経由）。紛失デバイスの捜索や、会場内の分布確認に使用できます。

Response — 200 OK 
{
  "lat": 21.5912,
  "lng": -158.1041,
  "accuracy_m": 15,
  "last_seen": "2026-07-15T14:32:00Z"
}
POST /api/v1/devices/:id/ring

デバイスの音を鳴らします。紛失時にデバイスを見つけるために使用します。

Response — 200 OK 
{
  "id": "SEED-0001",
  "ringing": true,
  "duration_s": 30
}

OTA更新 Over-the-Air Updates

POST /api/v1/ota/upload

ファームウェアバイナリをアップロードします。Content-Type: application/octet-stream でバイナリを送信してください。

cURL Example 
curl -X POST https://koe.live/api/v1/ota/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/octet-stream" \
  -H "X-Firmware-Version: 1.3.0" \
  --data-binary @firmware-v1.3.0.bin
Response — 201 Created 
{
  "version": "1.3.0",
  "size_bytes": 524288,
  "sha256": "a1b2c3..."
}
POST /api/v1/ota/deploy

アップロード済みファームウェアを全デバイスに配信します。デバイスは次回起動時に自動で更新を取得します。

Request Body 
{
  "version": "1.3.0",
  "rollout_percent": 100
}
Response — 200 OK 
{
  "deploy_id": "dep_xxx",
  "version": "1.3.0",
  "target_devices": 847,
  "status": "rolling_out"
}
GET /api/v1/ota/status

OTA更新の進行状況を取得します。

Response — 200 OK 
{
  "current_version": "1.3.0",
  "updated": 623,
  "pending": 224,
  "failed": 0,
  "progress_percent": 73.6
}

分析 Analytics

GET /api/v1/analytics/summary

利用統計サマリーを取得します。アクティブデバイス数、ピーク時間、平均再生時間、返却率を含みます。

Response — 200 OK 
{
  "active_devices": 847,
  "peak_hour": "16:00",
  "avg_play_time_h": 4.2,
  "return_rate": 0.973
}
GET /api/v1/analytics/heatmap

エリア別の接続密度データを取得します。会場マップの可視化に使用できます。

Response — 200 OK 
{
  "zones": [
    { "name": "メインステージ", "count": 523 },
    { "name": "サブステージ", "count": 198 },
    { "name": "フードエリア", "count": 126 }
  ]
}

レート制限 Rate Limits

プランに応じてリクエスト数の上限が設定されています。制限を超えた場合は 429 Too Many Requests が返されます。

プランリクエスト上限配信チャンネル数
スターター1,000 リクエスト/分3
プロ10,000 リクエスト/分20
エンタープライズ無制限無制限

SDKs

公式SDKを使えば、数行のコードでKoe APIを統合できます。

Node.js / TypeScript

npm install @koe/sdk

Python

pip install koe-sdk
Node.js Example 
import { Koe } from '@koe/sdk';

const koe = new Koe('YOUR_API_KEY');

// 配信を開始
const broadcast = await koe.broadcast.start({
  channel_name: 'メインステージ',
  codec: 'lc3',
  sample_rate: 48000
});

// デバイス一覧を取得
const devices = await koe.devices.list();
console.log(`接続中: ${devices.length} 台`);

フェスアプリ連携 Seedのステータス（バッテリー・チャンネル）をフェス公式アプリに表示。来場者がチャンネルを切り替え可能。
